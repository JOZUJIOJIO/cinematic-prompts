## ✨ 新增功能详解

### 1. 九大要素标注系统

**功能说明**:
参考行业标准分镜脚本规范,在 JSON 结构中新增"九大要素标注"字段,确保生成的分镜脚本符合专业制作要求。

**九大要素包含**:
```json
"九大要素标注": {
  "1_镜号": "Shot 001",
  "2_景别": "MCU/WS/CU等",
  "3_时长": "5秒",
  "4_内容": "画面主要内容描述",
  "5_运动": "镜头运动方式",
  "6_对白": "角色台词或旁白(如有)",
  "7_音效": "环境音/音乐提示",
  "8_转场": "Cut/Fade/Dissolve等",
  "9_备注": "导演意图/情绪重点"
}
```

**应用场景**:
- 与制片团队沟通时使用标准术语
- 导入剪辑软件时有完整的元数据
- 制作拍摄计划表时直接引用

**使用示例**:
```json
{
  "shot_number": 3,
  "九大要素标注": {
    "1_镜号": "Shot 003",
    "2_景别": "CU",
    "3_时长": "5秒",
    "4_内容": "宇航员扣紧头盔密封环",
    "5_运动": "静止镜头,焦点从手部转至眼睛",
    "6_对白": "无",
    "7_音效": "机械咔哒声,呼吸声",
    "8_转场": "Cut",
    "9_备注": "强调专业性和紧张感"
  }
}
```

### 2. 工作流时间参考

**功能说明**:
提供 6 个阶段的预估时间参考,帮助用户合理规划项目进度,同时与传统人工分镜对比,量化效率提升。

**时间估算表**:

| 阶段 | 预估时间 | 主要工作 |
|------|---------|----------|
| 需求分析与DNA创建 | 15-30分钟 | 分析参考图、创建DNA模板 |
| 分镜脚本设计 | 30-60分钟 | 拆解场景、设计构图、撰写提示词 |
| AI图片生成 | 1-3小时 | Midjourney/SD生成静态图、迭代优化 |
| 图转视频处理 | 1-2小时 | Runway/Pika添加运动、导出视频 |
| 分镜标注与导出 | 30-60分钟 | 九大要素标注、多格式导出 |
| 审核与迭代优化 | 30分钟-数小时 | 收集反馈、修改生成、版本管理 |

**总计**: 3-6小时(快速) / 1-2天(常规) / 2-4天(复杂)
**传统人工分镜**: 1-3周
**效率提升**: 约 70-85%

**重要说明**:
- 时间为参考值,实际因项目而异
- 不包括 AI 工具排队等待时间
- 首次使用需要额外学习时间
- 建议预留 20-30% 缓冲时间

### 3. 多格式导出指南

**功能说明**:
提供 6 种常用格式的导出指南,满足不同场景的使用需求,从导演审阅到技术开发全覆盖。

**支持格式**:

#### 📄 PDF 分镜脚本
- **用途**: 正式文档、打印审阅、存档
- **结构**: 封面→目录→DNA配置→分镜详情→技术规格
- **工具**: Typora, Pandoc, Adobe InDesign

#### 📊 PPT 演示文稿
- **用途**: 团队会议、客户提案、现场演示
- **特点**: 图文结合、视频嵌入、动画展示
- **工具**: PowerPoint, Keynote, Google Slides

#### 🎥 视频预览
- **用途**: 快速预览、社交分享、初步审片
- **内容**: 串联所有镜头、添加字幕、配背景音乐
- **工具**: Premiere Pro, CapCut, FFmpeg

#### 📱 Notion/飞书文档
- **用途**: 实时协作、在线评论、版本追踪
- **优势**: 团队成员可直接评论、移动端查看
- **结构**: 项目概述→DNA配置→分镜列表→反馈收集

#### 🔧 JSON 数据文件
- **用途**: 程序化处理、批量导入、二次开发
- **场景**: 导入剪辑软件、自动化生成、数据库存储

#### 📋 Excel 表格
- **用途**: 拍摄计划、预算估算、进度追踪
- **内容**: 镜号、演员、道具、设备、预算
- **工具**: Excel, Airtable, Notion Database

**导出优先级**:
- **必须**: Markdown + PNG 图片序列
- **推荐**: PDF + PPT + JSON
- **专业**: 以上全部 + 视频 + 在线文档 + Excel

### 4. 迭代优化与版本管理

**功能说明**:
建立完整的反馈循环和版本控制机制,支持多轮迭代和团队协作,确保分镜质量持续优化。

#### 🔄 5 步迭代循环

```
初版生成 → 提交审核 → 收集反馈 → 针对性修改 → 版本发布
     ↑                                            ↓
     └────────────────────────────────────────────┘
                    (循环直至终版)
```

#### 📊 反馈分类系统

- **A级(必改)**: 影响叙事逻辑、角色形象严重偏差
- **B级(建议改)**: 构图/色调/运动不够理想
- **C级(可选改)**: 细节优化、锦上添花

#### 📝 版本命名规范

采用语义化版本控制 (Semantic Versioning):

**格式**: `v主版本.次版本.修订号`

**示例**:
- `v1.0.0` - 初版交付
- `v1.0.1` - 修复镜头3的人物一致性(小修订)
- `v1.1.0` - 增加了2个新镜头(次版本更新)
- `v2.0.0` - 更换了整体视觉风格(主版本更新)

**文件命名**:
```
项目名称_分镜脚本_v1.0.0_2026-01-10.md
项目名称_分镜脚本_v1.0.1_2026-01-11.md
项目名称_分镜脚本_v2.0.0_2026-01-15.md
```

#### 📋 变更日志模板

```markdown
## 📝 版本历史

### v1.1.0 (2026-01-11)
**变更类型**: 次版本更新
**修改内容**:
- [修复] 镜头3人物发色一致性问题
- [优化] 镜头5月球表面光影对比度
- [新增] 镜头9观察窗特写场景
**修改镜头**: 3, 5, 新增9
**审核状态**: 待客户确认
```

#### 🛠️ 4 种修改策略

1. **单镜问题**(最常见)
   - 仅修改该镜头的变化层
   - 保持 DNA 不变
   - 重新生成该镜头

2. **一致性问题**
   - 修改全局 DNA 模板
   - 重新生成所有受影响镜头
   - 更新大版本号

3. **叙事结构调整**
   - 增删镜头或调整顺序
   - 重新编号所有镜头
   - 记录重大变更

4. **视觉风格调整**
   - 修改 style_DNA
   - 重新生成所有镜头
   - 视为新主版本

### 5. 最佳实践建议

**功能说明**:
总结实战经验,提供提高一致性、加速生成、避免错误、团队协作等方面的实用技巧。

#### ✅ 提高一致性的技巧
1. DNA 模板越详细越好 - 初期多花时间完善
2. 权重要果断 - 关键特征不低于 1.3
3. 参考图质量 - 高清、光线均匀效果最好
4. 首镜最关键 - 第一镜要迭代到完美

#### ⚡ 加速生成的技巧
1. 批量提示词 - 准备好后一次性提交
2. 使用种子值 - SD 可固定 seed 保持一致性
3. 模板复用 - 相似项目复用之前的 DNA
4. 快速预览 - 先低分辨率预览,确认后生成高清

#### ❌ 避免常见错误
1. 不要跳过 DNA 模板直接生成
2. 不要在提示词中重复描述固定元素
3. 不要频繁更换参考图
4. 不要忽视版本管理

#### 👥 团队协作建议
1. 明确分工 - DNA设计/生成/审核
2. 统一标准 - 使用相同的九大要素模板
3. 及时沟通 - 发现问题立即反馈
4. 知识共享 - 建立团队 DNA 模板库
